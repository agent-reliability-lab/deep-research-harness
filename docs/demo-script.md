## 旁白完整版(连读,约 400 字 / 3 分钟)

> 我的第一个真实评测 run,跑了满分 1.0。但这个视频不是讲这个 1.0 的——真正的故事,是它揪出的 4 个测量 bug。
>
> 2026 年 Agent 的瓶颈不是能力,是可靠性和单位经济。我做了一个 Agent 可靠性实验室,只回答一个问题:哪些可靠性控制真正提升任务成功率,代价多少?两个北极星——证据化任务成功率 EGTSR,和每个成功任务的成本 CPS。
>
> 在让模型产出任何一个数字之前,我先让它过四道资格门:身份不被静默替换、缓存账本对得上、工具调用穿过供应商不被改坏、20 次连发稳定。任一不过,不许计分。注意这里——G3、G4 各挂过一次,失败件我留作证据,没悄悄重跑。一道从不拦东西的门,不是门。
>
> 一条命令复现第一个真实 run。EGTSR 1.0,每个成功任务成本不到一分钱,98% 输入 token 是缓存命中——这正是 CPS 能压这么低的原因。Agent 是真的在冻结的官方文档里检索、记录有据证据、产出带引用的报告。
>
> 但 1.0 不是成绩——它是两任务 smoke,任务还是我设计成能过的。真正的价值是这次真 run 冲出来、fixture 永远冲不出来的 4 个测量 bug:markdown 清洗、rubric 假阴性、引用排版、trace 版本。这些不是 Agent 的 bug,是尺子本身的 bug。在评测里,修那把尺子,才是真正的工作。
